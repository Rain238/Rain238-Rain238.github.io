# Hexo 基本配置

  • 根目录下的 _config.yml , 就是 Hexo 框架的整个配置文件了，可以在里面修改大部分的配置，也可以去官方参考官方文档.
  • 我们来打开 _config.yml 文件，

    # Site 部分内容

# URL 部分

  • url 改成你的网站域名
  • permalink 文章的→ 永久链接 ←的格式有很多可去看官方介绍
  • 中间的默认就行，找到最后的 theme 这是配置主题的，例如你 theme 文件夹下面有 N 个主题有 A 和 B 主题，你想有 A 主题 theme 这个地方就填 A
  • deploy 是来和你的 GitHub 仓库关联的， type 填 Git， repository 填你的 GitHub 仓库的链接， branch 填 master

# Front-matter

  • Front-matter 是文件最上方以 --- 分隔的区域，用于指定个别文件的变量，例如
  • title 标题
  • date 日期
  • update 更新日期
  • categories 分类
  • tags 标签
  • comments 开启文章评论
  • placeholder 评论框占位符

# 布局 layout

# 不同主题可能会有所差异

  • layout 该文件夹在你的主题目录下

    # - 该目录下可能包含

    _alternate
    _macro
    _partials
    category:类别
    index:首页
    page:页
    post
    tag:标签
    等文件

    # - _partials 目录下可能包含

    head:头
    post
    sidebar:侧边栏
    third-party:第三方
    footer:页脚
    header:标题
    layout:布局
    pagination:分页

    # - post 目录下可能包含

    copyright:版权
    footer:页脚
    nav:导航栏
    post
    reward

    # _drafts 草稿

如果你写的文章不行在页面显示可以在根目录下的 source 的文件夹下创建一个 _drafts 这样的一个文件夹，然后把.md 放到这个文件夹下就可以了，这就是一个草稿文件夹

# 更换主题

主题下载→点击这里← 将下载好的主题解压到 themes 文件下，改一下 _config.yml 配置文件，不是改主题的 _config.yml ，将 themes 部分换成你下载的主题的名字

# 总结

# DIY 前请先备份 <- 鼠标选中查看

如果想自己来 DIY 主题，DIY 前请先备份，需要有前端知识以及 ejs 语法等其实多鼓捣鼓捣也就不难了
scripts：js 脚本
source：里面是 CSS 跟 JS 和图片文件
